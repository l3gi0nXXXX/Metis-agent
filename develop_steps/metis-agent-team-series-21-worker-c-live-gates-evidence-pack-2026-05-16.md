## Implemented Gate Contract

`scripts/agentteam-manual-acceptance-gate.sh` now emits:

- `report.json`
- `manual-acceptance-template.md`

The JSON report includes:

- series 21 source metadata;
- current git head and branch;
- Phase 0, Phase 3, Phase 4, Phase 5, and Phase 9 statuses;
- G01-G25 status rows;
- M01-M32 manual acceptance rows;
- external resource requirements;
- opt-in state for `METIS_AGENTTEAM_LIVE_TELEGRAM`, `METIS_AGENTTEAM_LIVE_FEISHU`, and `METIS_AGENTTEAM_CONTROL_UI_URL`;
- redaction scan status.

The gate records only `provided-redacted` or `missing` for live resource variables. It does not record raw Telegram, Feishu, OAuth, proxy, provider, or request credential values.

## Status Semantics

- `local-pass`: local script/source/test evidence is complete.
- `external-resource-required`: a real Telegram or Feishu test resource is needed, or live opt-in is disabled.
- `operator-record-required`: opt-in resources are present or a browser/operator runtime check is needed, but the script does not claim external success.

Default behavior does not call Telegram or Feishu network endpoints.
